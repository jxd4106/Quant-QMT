# V0.5 方案（四方评审后修订）

## 回应四方挑刺

### 🔴 致命级回应

| 挑刺 | 来源 | 处置 | 说明 |
|------|------|------|------|
| **无回测验证** | 战略 | **驳回（用户决策）** | 用户已确认 PRD 继续，回测数据不阻塞方案设计。但 PRD 中将标注信号胜率/回撤为「待实盘验证假设」 |
| **盘中信息不完整** | 工程+战略 | **接受** | 新增设计：信号判定只在每日 **14:55** 执行一次（而非每 tick）。此时日线信息已接近完整，剩余 5 分钟限价单有足够时间成交 |
| **偏度/多股同时止损** | 商业+战略 | **接受（用户决策）** | 止损线从 30% 降至 **15%**（用户拍板）。新增：单日触发 2 次止损 → 当日暂停所有交易 |
| **港股同一套信号未验证** | 商业+战略 | **驳回（用户决策）** | 用户确认港股不改参数，用同一套信号。PRD 中标注为「港股信号参数待实盘校准」 |
| **多股仓位无总量上限** | 商业 | **接受** | 新增 F15：全局仓位上限（各股仓位权重之和 × 50% 不超过 1.0，超出则按比例归一化） |
| **防重复机制未定义** | 工程 | **接受** | 信号判定统一在每日 **14:55** 执行（一次），天然防重复 |
| **止损恢复流程空白** | 设计 | **接受** | 新增：`state.json` 同目录放 `override.json`，用户只需写 `["603501.SH"]` 即可恢复 |
| **手动交易 vs 策略状态不对账** | 设计 | **接受** | 新增 F16：启动时查询券商实际持仓，与 state.json 对账。不一致则以券商为准，打告警日志 |
| **单点故障/无告警** | 商业 | **部分接受** | 新增：心跳写文件 + 外部 watchdog 脚本建议（非必须，PRD 附录）。策略内加异常计数器，连续 N 次异常打印醒目告警 |

### 🟠 严重级回应

| 挑刺 | 来源 | 处置 | 说明 |
|------|------|------|------|
| **市场状态识别缺失** | 战略 | **接受** | 新增 F17：MA20 斜率过滤——MA20 斜率向下时，仅执行卖出信号，不执行买入 |
| **港股 T+2 交收 vs A 股 T+1** | 商业 | **接受** | 下单前增加「实际可用资金」检查（`query_stock_asset.cash` 而非自己算）。资金不足则跳过该股买入 |
| **运维连续性** | 商业 | **驳回** | 个人使用，无 SaaS 运维需求。日志 + 告警足够 |
| **ContextInfo/G 对象清单缺失** | 工程 | **接受** | 见下方「G 对象状态清单」 |
| **miniQMT vs QMT get_market_data** | 工程 | **接受** | 仅支持 QMT 内置模式，不做 miniQMT 兼容（简化） |
| **get_full_tick 延迟/OOM** | 工程 | **部分接受** | 逐股调用 `get_full_tick([stock_code])` 而非全市场模式，单次 < 50ms，无视 OOM 风险 |
| **配置体验** | 设计 | **驳回（用户决策）** | 用户此前已确认「代码内置字典」。到 MVP 完成后再考虑 JSON 配置 |

| **try-except 静默失败** | 设计 | **接受** | 统一定义：API 异常 → 写 ERROR 日志 + 同类错误 5 分钟限频 + 连续 3 次不同类异常打印控制台告警 |
| **xttrader 兼容层** | 工程 | **接受** | 仅针对国金 QMT。硬编码 `from xtquant.xttrader import XtQuantTrader`，不做多版本兼容 |
| **历史数据 get_market_data 调用方式** | 工程 | **接受** | 使用 `ContextInfo.get_market_data_ex()`（QMT 内置环境推荐方式） |
| **港股代码后缀矛盾** | 工程 | **接受** | 确认使用 `.HGT`（沪港通）和 `.SGT`（深港通），在股票池配置中加注释说明 |

## G 对象状态清单（工程 Reviewer 要求）

```python
class G:
    # 跨 tick 持久化（存 G）
    trade_count_today: dict   # {stock_code: int}
    sold_today: dict          # {stock_code: bool}  当日卖出黑名单
    stop_loss_triggered: dict # {stock_code: bool}
    last_signal_time: str     # "2026-07-02 14:55"  防重复判定时间戳
    error_count: dict         # {error_type: int}    异常计数器
    daily_stop_count: int     # 当日止损触发次数（≥2 熔断）
    heartbeat_time: str       # 最后心跳时间

    # 不存 G（每次 handlebar 实时取）
    # - get_full_tick() 返回的实时价格/行情
    # - query_stock_position() 返回的持仓
    # - query_stock_asset() 返回的资金
    # 这些每次实时调用 xtdata/xttrader，不缓存在任何对象中
```

## 信号判定时机（工程 Reviewer 要求）

```
14:50-14:55 → 不做判定，只更新历史数据缓存
14:55:00   → 执行一次全股票池信号判定 + 下单
14:55-15:00 → 不再下单，只检查订单成交状态 + 写日志

港股适配：
- 港股早市收盘前 5 分钟（11:55）也执行一次判定
- 港股午市收盘前 5 分钟（15:55）执行一次判定
```
