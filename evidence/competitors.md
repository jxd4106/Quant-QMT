# 半仓滚动 QMT · 竞品与行业调研

## 1. QMT 平台 API 与约束

### 关键发现
- QMT 正确模块名是 `xtquant.xttrader`，**不存在 `xtquant.xtrading` 模块**——现有代码的 try/import 兼容层可能在实盘全部失败
- `handlebar` 在实盘中**每个 tick（约 3 秒）触发一次**，不是每根 K 线一次
- **ContextInfo 回滚机制**：逐 K 线模式下 ContextInfo 变量修改会被回滚，必须用全局变量存储状态
- `get_market_data` 返回 `dict[字段 → DataFrame]`，DataFrame index 为股票代码
- `get_full_tick` 更新频率约 3 秒，全市场模式约 80MB 内存
- 港股代码格式：5 位 + `.HGT`（沪港通）/ `.SGT`（深港通），非 `.HK`
- 港股手数 `VolumeMultiple` 必须通过 `get_instrument_detail` 查询

### QMT 策略框架关键约束
- `init()` 执行一次，`handlebar()` 每个 tick 触发
- quickTrade 参数：0=K线结束生效，1=最新K线立即，2=任何情况立即
- **策略 F5 运行无法下单**，必须在"模型交易"界面以实盘模式运行
- 所有策略在单一线程运行，禁止 `time.sleep()`
- **`download_history_data` 不支持 callback**，仅 `download_history_data2` 支持

## 2. 止损行业实践

### 核心结论
- 量化社区主流止损区间：**5%-15%**，以 8% 为最常见默认值
- **30% 止损在量化中属于异常宽松**（牛津论文使用 25% 为"宽松基线"）
- 业界推荐：ATR 动态止损（减少最大回撤约 14.7%）优于固定比例止损
- **个股止损 + 账户熔断双层风控是强制要求**
- 跌 30% 需涨 43% 回本，非线性风险

## 3. 港股 + A 股混合策略

### 关键发现
- 港股交易时间：早市 9:30-12:00（比 A 股多半小时），午市 13:00-16:00（多 1 小时）
- 港股通综合费率约为 **A 股 3-4 倍**（印花税双边 0.1%）
- 港股通代码格式：5 位 + `.HKSH`/`.HKSZ`，非 `.HK`
- 港股 T+0 交易但 **T+2 交收**，需追踪 2 天资金链
- **港股通不支持市价单**，需用 `LATEST_PRICE` 模拟
- 港股交易日历与 A 股不同（佛诞、圣诞等）
- 买入时参考汇率多冻结约 3% 汇差，T+2 清算后释放

## 4. QMT 实盘工程实践

### P0 致命风险
1. **ContextInfo 回滚**：变量修改会丢失
2. **deal_callback 重复推送**：断线重连后成交重复推送
3. **quickTrade 参数误用**：日线设 0 导致全天不下单
4. **止损后被反手买入**：形成死循环
5. **运行模式选错**：F5 无法下单

### 工程建议
- 实盘上线前：模拟信号 1-2 周 → 模拟柜台 → 小资金 3-7 天 → 逐步加仓
- 日志记录 7 个关键节点：初始化/行情订阅/触发条件/委托详情/成交反馈/资金快照/异常
- 推荐长期挂机用 miniQMT（资源占用约 30%，支持外部 IDE）
- 当日卖出标的需加入黑名单防止止损后反手买入
