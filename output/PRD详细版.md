## 1. 产品定位与价值

### 一句话定位

在国金 QMT 平台上运行的自动化半仓滚动量化交易策略——6 信号技术指标体系，A 股 + 港股通双市场，限价单执行，单股 15% 止损。

### 核心价值

> 不是有没有策略的问题——是策略能不能无情绪、全天候、跨市场地落地执行。

- **纪律化执行**：每日定时统一判定信号，消除人工盯盘的情绪干扰
- **跨市场覆盖**：A 股 + 港股通双市场，策略内按各自交易时段自动适配
- **半仓滚动**：凯利公式在特定胜率/赔率组合下的数学最优解，震荡市中兼顾攻守
- **可恢复**：止损触发后一键恢复，策略重启后状态自动对账

### 价值论证

| 维度 | 论证 |
|------|------|
| 谁在付代价 | 策略操作者本人——每日手工盯盘 2-3 小时，信号判断受情绪影响 |
| 代价多大 | 按 10-50 万资金量级，纪律化执行比手工操作保守估计年化差 5-10 个百分点 |
| 解决后值多少 | 省时间 + 提收益 + 防深套，三者叠加对个人交易者有实质意义 |
| 最锋利的一句话 | 策略能不能无情绪、全天候、跨市场地落地执行 |

## 2. 用户与场景

### 用户

策略开发者本人（唯一用户）。负责：配置股票池 → 启动策略 → 查看日志 → 止损后手动恢复。

### 核心场景

> 每个交易日盘中，QMT 自动维护股票池中每只个股的历史数据。收盘前 5 分钟统一对全股票池执行信号判定，限价单下单。卖出优先于买入。每只个股独立做半仓滚动——目标仓位 = 该股分配权益 × 50%。

### 典型交易日时间线

```text
08:30  策略自动启动，下载历史数据，对账券商持仓
09:30  A 股开盘 + 港股早市开盘
11:30  A 股午休
11:55  港股早市收盘前信号判定（仅港股）
12:00  港股午休
13:00  A 股午市开盘 + 港股午市开盘
14:55  ⭐ A 股信号统一判定 + 下单
15:00  A 股收盘
15:55  ⭐ 港股午市收盘前信号判定 + 下单（仅港股）
16:00  港股收盘
16:10  收盘快照 + 日志摘要
```

### 交易时段过滤

| 市场 | 早市 | 午市 | 信号判定时间 |
|------|------|------|-------------|
| A 股 | 9:30-11:30 | 13:00-15:00 | 14:55 |
| 港股通 | 9:30-12:00 | 13:00-16:00 | 11:55 + 15:55 |

## 3. 核心功能需求

### F1 · 股票池配置

```python
STOCK_POOL = {
    '603501.SH':   {'name': '韦尔股份', 'weight': 1.0},
    '00700.HGT':   {'name': '腾讯控股', 'weight': 0.5},
    '01810.SGT':   {'name': '小米集团', 'weight': 0.5},
}
```

- 每只股票指定 `weight`（权益分配权重）
- A 股后缀 `.SH` / `.SZ`
- 港股通后缀 `.HGT`（沪港通）/ `.SGT`（深港通）
- 代码内 Python 字典，改配置需改代码（V0 阶段）

### F3 · 半仓滚动

```text
单股目标仓位 = account_total_asset × weight × 0.5
单股目标买入量 = max(目标仓位股数 - 当前持仓股数, 0)
单股目标卖出量 = 当前持仓股数 // 2  （每次卖 50% 持仓）
```

### F15 · 全局仓位上限

```text
总目标仓位 = Σ(weight_i × 0.5)
若总目标仓位 > 1.0，各股目标仓位按比例归一化
```

> 每只股的 weight 之和建议 ≤ 2.0（即总仓位 ≤ 100%）。实际超出时自动归一化，不报错。

### F4 · 限价单执行

| 方向 | 价格 | 价格类型 |
|------|------|---------|
| 卖出 | `current_price × 0.998`（向下 0.2%） | `FIX_PRICE` |
| 买入 | `current_price × 1.002`（向上 0.2%） | `FIX_PRICE` |

### F19 · 信号判定时机

信号判定统一在每日**对应市场收盘前 5 分钟**执行一次（而非每 tick 判定）。

| 市场 | 判定时间 | 理由 |
|------|---------|------|
| A 股 | 14:55 | 日线数据已接近完整 |
| 港股早市 | 11:55 | 早市收盘前 |
| 港股午市 | 15:55 | 午市收盘前 |

14:50-14:55 期间只更新历史数据缓存，不做判定。14:55 执行一次全股票池信号扫描，14:55-15:00 不再下单。

### F9 · 卖出优先于买入

同只股票同一次判定中，卖出信号优先于买入信号。卖出了当天不再买入该股。

## 4. 策略信号体系

### 信号定义

#### 卖出信号（S1 > S2 > S3 优先级）

| 信号 | 名称 | 条件 | 动作 |
|------|------|------|------|
| **S1** | 破位卖出 | 收盘价从 MA20 之上掉到 MA20 之下 | 卖出当前持仓的 50% |
| **S2** | 量价背离卖出 | 创 20 日新高 且 成交量 ≤ 5 日均量 × 0.7 | 卖出当前持仓的 50% |
| **S3** | 放量长上影卖出 | 成交量 ≥ 5 日均量 × 1.5 且 长上影线 且 当日收跌 | 卖出当前持仓的 50% |

#### 买入信号（B1 > B2 > B3 优先级）

| 信号 | 名称 | 条件 | 动作 |
|------|------|------|------|
| **B1** | 突破买入 | 收盘价从 MA20 之下站上 MA20 之上 | 买入到目标仓位 |
| **B2** | 长下影买入 | 长下影线 且 成交量 ≥ 5 日均量 × 1.5 | 买入到目标仓位 × 1/3 |
| **B3** | 放量三线突破 | 阳线 且 收盘站上 MA5/MA10/MA20 且 成交量 ≥ 5 日均量 × 2.0 | 买入到目标仓位 |

### 信号参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `VOL_RATIO_LOW` | 0.7 | S2 缩量阈值 |
| `VOL_RATIO_MID` | 1.5 | S3/B2 放量阈值 |
| `VOL_RATIO_HIGH` | 2.0 | B3 强放量阈值 |
| `SHADOW_RATIO` | 2.0 | 影线/实体比 |
| `SHADOW_PCT` | 0.40 | 影线/振幅比 |
| `TARGET_PCT` | 0.5 | 目标仓位比例 |
| `B2_RATIO` | 1/3 | B2 买入比例 |

### 信号理论基础

| 信号 | 对应经典形态 | 技术分析依据 |
|------|-------------|-------------|
| S1 破位卖出 | MA20 支撑破位 | 中期趋势转弱，20 日均线是 A 股最广泛使用的趋势线 |
| S2 量价背离 | 顶部缩量 | 创新高但无成交量配合 = 多头力量衰竭 |
| S3 长上影+放量 | 射击之星 | 冲高回落 + 放量 = 主力出货痕迹 |
| B1 突破买入 | MA20 突破 | 中期趋势转强，站上 MA20 = 空转多确认 |
| B2 长下影+放量 | 锤子线 | 探底回升 + 放量 = 底部承接有力 |
| B3 三线突破+放量 | 强势突破 | 一阳穿三线 + 倍量 = 趋势启动信号 |

## 5. 技术指标计算

### 指标列表

| 指标 | 计算方式 | 用途 |
|------|---------|------|
| MA5 | 5 日收盘价简单移动平均 | 短期趋势 |
| MA10 | 10 日收盘价简单移动平均 | 中期趋势 |
| MA20 | 20 日收盘价简单移动平均 | 趋势基准线 |
| vol_ma5 | 5 日成交量简单移动平均 | 量能基准 |
| vol_ratio | 当日量 / vol_ma5 | 放量/缩量判定 |
| body_abs | `abs(close - open)` | 实体长度 |
| range_ | `high - low` | 全日振幅 |
| upper_shadow | `high - max(open, close)` | 上影线长度 |
| lower_shadow | `min(open, close) - low` | 下影线长度 |
| long_upper | 上影 > 实体 × 2 且 > 振幅 × 40% | S3 判定 |
| long_lower | 下影 > 实体 × 2 且 > 振幅 × 40% | B2 判定 |
| ma_bull | MA5 > MA10 > MA20 | 多头排列 |
| new_high20 | 当日最高价 > 前 20 日最高价 | S2 判定 |
| break_3ma | 收盘站上 MA5/MA10/MA20 + 阳线 | B3 判定 |

### 实现约束

- 纯 numpy 实现，不依赖 pandas（QMT 内置 Python 可能版本不兼容）
- 历史数据取 60 根日线，不足 60 跳过该股当日
- 当前日线追加到历史数组末尾再计算指标

## 6. 数据架构

### 行情数据流

```text
init() 阶段：
  xtdata.download_history_data(stock, period='1d')  → 下载历史日线
  ContextInfo.get_market_data_ex(...)                → 获取 60 根日线到内存缓存

handlebar() 阶段：
  ContextInfo.get_full_tick([stock_code])            → 获取实时价格
  追加当前 bar 到历史数组末尾
  计算指标 → 判定信号 → 下单
```

### 账户数据流

```text
xttrader.query_stock_asset(account)                  → 总资产 / 可用资金
xttrader.query_stock_position(account, stock_code)   → 持仓数量 / 成本价 / 盈亏比

不缓存——每次判定时实时查询
```

### 状态持久化

```json
{
  "date": "2026-07-02",
  "stocks": {
    "603501.SH": {
      "stop_loss_triggered": false,
      "trade_count_today": 0,
      "sold_today": false,
      "stop_loss_base": null
    }
  },
  "daily_stop_count": 0
}
```

- 文件位置：策略 `.py` 同目录
- 跨日自动重置 `trade_count_today` / `sold_today` / `daily_stop_count`
- `stop_loss_triggered` 跨日**不**重置（持久暂停，需手动恢复）
- `stop_loss_base`：首次启动时从 `query_stock_position.avg_price` 获取并持久化

### 对账机制（F16 · 启动时执行）

```text
策略启动时，对每只股票：
  1. 查询券商实际持仓数量（xttrader.query_stock_position）
  2. 与 state.json 记录的预期持仓对比
  3. 不一致 → 以券商实际持仓为准，更新 state.json，打印 WARNING 日志
```

## 7. QMT 框架适配

### 运行环境

- **平台**：国金 QMT（内置模式）
- **Python**：3.6-3.12（64 位）
- **依赖**：`xtquant`（QMT 自带）、`numpy`
- **不兼容 miniQMT**（V0 仅支持 QMT 内置模式）

### 模块导入

```python
from xtquant import xtdata
from xtquant.xttrader import XtQuantTrader
from xtquant.xtconstant import *
```

不做多版本兼容——仅针对国金 QMT 当前版本。

### handlebar 适配

- handlebar 实盘中每约 3 秒触发一次
- **不在 handlebar 内逐 tick 判定信号**
- handlebar 内只做：检查当前时间 → 若到达判定时间点（14:55/11:55/15:55）→ 执行一次全股票池信号判定 + 下单 → 标记已判定
- 防重复：用全局 `G.last_signal_time` 记录上次判定时间戳，同一分钟不重复判定

### ContextInfo 回滚防护

所有跨 tick 持久化状态存全局 `class G` 对象，**不存 ContextInfo 属性**：

```python
class G:
    trade_count_today: dict
    sold_today: dict
    stop_loss_triggered: dict
    last_signal_time: str
    daily_stop_count: int
```

实时行情/持仓/资金每次 handlebar **实时调用 xtdata/xttrader 获取**，不缓存。

### 下单参数

```python
passorder(
    opType=23,              # 23=买入 24=卖出
    orderType=1101,         # 单股普通股
    accountid=accID,
    stock_code=stock_code,
    prType=11,              # 指定价（限价）
    price=limit_price,
    volume=qty,
    strategyName='V7',
    quickTrade=1,           # 最新 K 线立即触发
    orderId=order_id,
    ctx=ContextInfo
)
```

## 9. 非功能需求

### 日志规范（F10）

| 项目 | 规格 |
|------|------|
| 位置 | 策略 `.py` 同目录 `logs/V7_YYYY-MM-DD.log` |
| 格式 | `时间 | 级别 | 股票代码 | 消息` |
| 级别 | DEBUG（无变化）/ INFO（交易决策）/ WARNING（数据缺失）/ ERROR（API 异常） |
| 轮转 | 每天一个文件，保留 30 天 |
| 控制台 | INFO 及以上同步输出到 QMT 控制台 |

### 日志记录的 7 个关键节点

1. 策略初始化完成
2. 历史数据下载状态（成功/失败/重试）
3. 信号判定时的关键参数值（价格、指标值、信号结果）
4. 委托单详情（股票、数量、价格、订单 ID）
5. 成交反馈（实际成交价、成交量）
6. 止损触发事件
7. API 异常/错误

### 运行心跳

每分钟 QMT 控制台输出一行状态摘要：

```text
[14:30] 603501.SH pos=1000 PnL=+2.1% | 00700.HGT pos=0 | 股票池 2 只 正常运行
```

### 错误处理策略

| 错误类型 | 处理 |
|----------|------|
| 数据下载失败 | init 阶段重试 3 次，间隔 2 秒。3 次失败跳过该股 |
| API 返回 None/空 | 该股当日跳过，写 WARNING 日志 |
| 下单失败（返回 None） | 写 ERROR 日志，该股当日不再交易 |
| 同类错误频繁出现 | 5 分钟内最多 3 条 ERROR 日志（限频） |
| 连续 5 次不同类异常 | QMT 控制台打印醒目的告警分隔线 |

### 性能要求

| 指标 | 目标 |
|------|------|
| 单次 handlebar 耗时 | < 100ms（不阻塞下个 tick） |
| 14:55 全股票池判定 | < 1s |
| 股票池容量 | ≤ 10 只 |
| 历史数据缓存 | 60 根日线/股 |

## 11. 运维与监控

### 日常运维

| 频率 | 操作 | 说明 |
|------|------|------|
| 每日盘后 | 查看 `logs/V7_YYYY-MM-DD.log` | 确认信号判定和下单记录 |
| 每周 | 复盘日志，检查信号触发频率是否异常 | — |
| 每月 | 清理 30 天前的日志文件 | 自动轮转 |
| 止损触发时 | 创建 `override.json` 恢复 | 确认触发原因后再恢复 |

### 告警策略

V0 阶段不依赖外部通知（无微信/钉钉推送）。出问题时依赖：

1. QMT 控制台醒目的 ERROR 日志（连续异常时打印分隔线）
2. 日志文件完整记录
3. 建议外挂看门狗脚本（附录）

### 单点故障须知

策略依赖一台 Windows PC 持续运行 QMT。以下场景会导致策略中断：

- Windows 更新自动重启
- 网络断线
- QMT 进程崩溃
- 券商服务器维护

**缓解措施**：状态文件持久化 + 启动对账，重启后可恢复。但盘中崩溃时当日剩余交易无法执行。

import os, time, json

### C. 故障排查 SOP（10 个常见错误）

| # | 症状 | 排查 | 解决 |
|---|------|------|------|
| 1 | 策略加载失败 | QMT 控制台错误信息 | 检查 Python 语法、numpy 是否安装 |
| 2 | 下载数据失败 | 日志显示 download 错误 | 检查网络、QMT 是否已登录、是否是交易时段 |
| 3 | 港股数据为空 | 日志打印 no history data | QMT 操作→数据管理→补充港股通历史行情 |
| 4 | 下单不成交 | 订单已发但未成交 | 检查限价单价格是否合理、该股是否涨跌停 |
| 5 | 可用资金不足 | 日志 insufficient cash | 检查全局仓位上限是否超限、T+2 交收是否到账 |
| 6 | 状态文件损坏 | 策略启动报 JSON 解析错误 | 删除 `state.json` 让策略重建（止损标记会丢失） |
| 7 | 信号反复触发 | 同一天多次打印信号日志 | 正常——14:55 单次判定天然防重复 |
| 8 | handlebar 不触发 | 控制台无输出 | 检查是否在模型交易→实盘模式运行（非 F5） |
| 9 | 港股代码报错 | 日志显示 code not found | 确认后缀是 `.HGT`/`.SGT` 而非 `.HK` |
| 10 | QMT 进程崩溃 | 控制台消失 | 重启 QMT，策略会自动对账恢复 |
