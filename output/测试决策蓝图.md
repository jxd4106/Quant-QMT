# 测试决策蓝图（已与用户逐轮确认）

## 1 测试范围
- 选中层：单元层 + 边界异常层 + QMT mock 集成层 + 验收契约（共 4 层）
- 阶段边界：仅 MVP（PRD §0 阶段一）
- 不适配跳过：组件/页面/E2E/安全鉴权/无障碍/性能（无 GUI/无 API/无数据库）

## 2 各层测试重点

### 单元层（纯函数，脱离 QMT 本地跑）
5 个模块，用 numpy 模拟行情数据：
- **技术指标计算**：15 个指标（MA5/10/20, vol_ratio, body_abs, range_, upper/lower_shadow, long_upper/lower, ma_bull, new_high20, break_3ma, close_ab_ma20 等），已知输入验证输出
- **6 信号判定**：S1/S2/S3/B1/B2/B3，正常触发 + 边界不触发 + 信号优先级
- **仓位数学**：目标仓位计算、全局归一化（>1.0 缩放）、手数对齐、B2 1/3 比例
- **限价单价格**：卖=close×0.998、买=close×1.002、round 到 2 位
- **交易时段过滤**：A 股时段(9:30-11:30,13:00-15:00)、港股时段(9:30-12:00,13:00-16:00)、判定时间点匹配

### 边界异常层（数据/状态容错）
7 类边界：
- state.json 损坏 → 重建默认状态
- state.json 缺失 → 首次初始化
- state.json 字段缺失 → 补默认值
- override.json 格式错误 → 跳过并告警
- 历史数据 < 60 根 → 该股当日跳过
- 行情返回 None/空 → 跳过不崩溃
- 可用资金不足 → 跳过买入不报错
- 持仓=0 卖出 → 跳过不报错
- 跨日重置：trade_count_today/sold_today/daily_stop_count 归零，stop_loss_triggered 保留

### QMT mock 集成层（mock xtdata/xttrader）
6 条完整链路：
- init 启动：下载数据 + 加载 state + 对账 + 摘要
- handlebar 非信号时段：时间不对时提前 return
- 14:55 全流程：行情→指标→信号→下单→日志
- 止损触发：profit_rate ≤ -15% → 标记 → 持久化 → 当日不再交易
- override.json 恢复：启动时读取 → 重置标记 → 删除文件
- 启动对账：券商持仓 vs state 不一致 → 以券商为准 → WARNING 日志

## 3 验收门禁（GOAL 终止条件）
- **完成定义**：三层全部 P0 + P1 用例转绿 = 交付。P2 尽力。
- **覆盖率目标**：单元层 >90%（核心算法）、边界异常层 >80%、集成层 >70%
- **变异测试**：不做（V0 阶段）
- **安全层特例**：不适用（无登录/无鉴权）
